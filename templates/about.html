What is CyberVantage?

CyberVantage is an innovative educational web platform designed to teach students about cybersecurity, phishing attacks, spam detection, and cyber safety through interactive, hands-on learning experiences. Our platform combines cutting-edge AI technology with proven educational methodologies to create engaging simulations that prepare students for real-world cyber threats.

Built with modern web technologies and powered by advanced AI models, CyberVantage provides a safe, controlled environment where students can learn to identify and respond to cyber threats without any real-world risk.

Australian Essential Eight Compliance

CyberVantage aligns with the Australian Cyber Security Centre's (ACSC) Essential Eight Maturity Model, a prioritized list of mitigation strategies to protect organizations against cyber threats. Our platform incorporates these principles to ensure robust security and educate users about these critical controls.

1. Application Control

Implemented

Only approved applications run on the platform. User-uploaded content is strictly validated and sandboxed to prevent malicious code execution.

2. Patch Applications

Implemented

All dependencies are regularly updated. Security patches are applied promptly to address vulnerabilities in Flask, SQLAlchemy, and other libraries.

3. Configure Microsoft Office Macros

Educational

While not directly applicable to our web platform, students learn about macro-based threats through our educational content and simulations.

4. User Application Hardening

Implemented

Web browsers security features are leveraged. Content Security Policy (CSP) headers and XSS protection mechanisms are enabled.

5. Restrict Administrative Privileges

Implemented

Role-based access control (RBAC) ensures users only have necessary permissions. Administrative functions are restricted to verified admin accounts.

6. Patch Operating Systems

Implemented

Platform runs on regularly updated infrastructure. Operating system patches are applied as part of our deployment process.

7. Multi-Factor Authentication

Roadmap

Currently uses JWT-based authentication. MFA implementation is planned for future releases to add an extra layer of security.

8. Regular Backups

Implemented

Database backups are performed regularly. Encrypted backups ensure data can be recovered in case of system failure or security incident.

Security Measures

CyberVantage implements multiple layers of security to protect user data and provide a safe learning environment:

Authentication & Access Control

  • JWT Authentication: Secure token-based authentication with HS256 algorithm
  • Password Security: BCrypt hashing with strong password requirements (8+ chars, mixed case, numbers)
  • Rate Limiting: Protection against brute force attacks (5 login attempts per 5 minutes)
  • Session Management: Secure server-side session storage with 1-hour token expiration

Data Protection

  • Encryption at Rest: Sensitive data encrypted using Fernet (AES-128) encryption
  • Secure Database: SQLAlchemy ORM prevents SQL injection attacks
  • Environment-based Secrets: No hardcoded credentials in source code
  • HTTPS Support: SSL/TLS encryption for data in transit (production)

Input Validation & Output Encoding

  • CSRF Protection: Flask-WTF CSRF tokens on all forms
  • XSS Prevention: Template auto-escaping and Content Security Policy headers
  • Email Validation: RFC-compliant validation using email-validator library
  • Input Sanitization: All user inputs validated and sanitized server-side

Security Headers

  • X-Frame-Options: DENY - Prevents clickjacking attacks
  • X-Content-Type-Options: nosniff - Prevents MIME type confusion
  • X-XSS-Protection: 1; mode=block - Browser-level XSS filtering
  • Referrer-Policy: same-origin - Controls referrer information

AI Safety

  • Multi-Provider Fallback: Azure OpenAI and Google Gemini with automatic failover
  • Content Filtering: AI responses validated to ensure appropriate educational content
  • Rate Limiting: API usage limits prevent abuse and manage costs
  • Prompt Engineering: Carefully designed prompts ensure safe, relevant AI outputs

Our Technology Stack

CyberVantage is built using modern, secure technologies:

  • Backend: Python Flask framework with SQLAlchemy ORM
  • Frontend: HTML5, CSS3, JavaScript with Jinja2 templating
  • AI Integration: Azure OpenAI and Google Gemini APIs
  • Database: PostgreSQL (production) / SQLite (development)
  • Security Libraries: BCrypt, PyJWT, Cryptography, Flask-WTF
  • Deployment: Vercel (serverless) with environment-based configuration
